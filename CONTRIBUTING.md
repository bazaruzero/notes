# Conventions

## Project structure

```
blog-markdown/
├── README.md                 # landing page (auto-blocks: pinned, latest, categories)
├── CONTRIBUTING.md           # this file
├── scripts/
│   └── new_post.py           # post creation + index rebuild
├── categories/
│   └── <name>.md             # one file per category
└── posts/
    └── <slug>/               # post folder
        ├── <slug>.md         # post content
        ├── images/           # post images
        └── files/            # auxiliary files and artifacts
```

## Naming conventions

- **Post slug**: `kebab-case`, only lowercase letters `a-z`, digits `0-9` and hyphens.
  Example: `postgresql-pitr-backup`.
- **`.md` filename** must match the folder name: `posts/foo-bar/foo-bar.md`.
- **Categories**: a single lowercase word (`postgresql`, `oracle`, `tools`).
- **Images and files**: arbitrary names; links to them are relative to the `.md`
  file: `![](images/diagram.png)`, `[script](files/run.sh)`.

## Frontmatter

Every post starts with a YAML block enclosed between `---` markers and wrapped
in an HTML comment `<!-- ... -->` so that GitHub/GitLab does not render the
metadata. The wrapper is required — the parser rejects unwrapped frontmatter.

```markdown
<!--
---
title: "Post title"         # required
slug: my-post-slug          # must match the folder name
created: 2026-07-21         # creation date
updated: 2026-07-21         # last edit date
author: admin
categories: [postgresql]    # list, multiple allowed
tags: [pitr, wal]           # free-form tags (not indexed yet)
pinned: false               # true → post appears in "Pinned posts"
description: "Short description for previews."
---
-->
```

## Auto-blocks

`README.md` and `categories/*.md` contain markers:

```
<!-- AUTO:XXX:BEGIN -->
...content is generated by the script...
<!-- AUTO:XXX:END -->
```

**Rule**: anything between the markers is overwritten on `--rebuild`. Do not edit
it manually — changes will be lost. Text **above** the markers in
`categories/*.md` can and should be edited (this is where the category
description lives).

## Creating a post

```bash
python scripts/new_post.py <slug> \
    --title "..." \
    --categories cat1,cat2 \
    [--pinned] [--author admin] [--description "..."]
```

The script will:
1. Create `posts/<slug>/<slug>.md` from the template.
2. Create `posts/<slug>/images/` and `posts/<slug>/files/` with a `.gitkeep`.
3. Rebuild indexes: `README.md` + all `categories/*.md`.
4. If the post references a non-existent category, create the category file.

## Rebuilding indexes without creating a post

```bash
python scripts/new_post.py --rebuild
```

Run this after any change to a post's frontmatter (date, `pinned`, categories).
